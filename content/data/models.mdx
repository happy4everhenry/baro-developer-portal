# Data Models

<PageMeta status="live" layer="frontend" />

<InfoBox type="info" title="What are data models?">
Data models are like blueprints for information. Just like a blueprint tells builders what a house should look like, a data model tells the app what shape the data should be. Every stock, every price update, and every chart point follows its own blueprint.
</InfoBox>

All models are `Sendable` and `Identifiable` -- meaning they are safe to pass between tasks and each one has a unique ID.

## Core Models

These are the main models that power most of the app.

<Accordion>
<AccordionItem title="Stock" icon="💹" badge="Core" defaultOpen={true}>

The main stock model used in lists, carousels, and search results.

```swift filename="Stock.swift"
struct Stock: Identifiable, Sendable {
    let id: String
    let rank: Int
    let name: String
    let code: String
    let exchange: StockExchange
    let marketCap: String
    let price: Int
    let change: Int
    let changePercent: Double
    let volume: Int64
    let dividendYield: Double
    let logoStyle: LogoStyle
    let isNXT: Bool
    let isFractional: Bool

    var isPositive: Bool { change >= 0 }
    var tradingValue: Int64 { Int64(price) * volume }
}
```

<InfoBox type="fun" title="How IDs work">
Each stock's `id` is a combination of its code and venue, like `"005930-combined"`. This way, the same stock on different venues has a different identity.
</InfoBox>

```swift filename="StockDetailInfo.swift"
struct StockDetailInfo: Identifiable, Sendable {

    // Fundamentals
    let per: Double, eps: Int, pbr: Double, bps: Int, dividendPerShare: Int

    // Chart
    let chartData: [ChartPoint], marketStatus: String, nxtStatus: String

    // Extended sections
    let investorFlow: InvestorFlow       // Foreign/institution/individual net
    let analystOpinion: AnalystOpinion   // Buy/hold/sell + broker opinions
    let reports: [AnalystReport]         // 12+ broker research reports
    let dividends: [DividendRecord]      // Yearly dividends
    let dividendYearGroups: [DividendYearGroup] // Quarterly breakdown
    let financials: [FinancialRecord]    // Summary financials
    let quarterlyFinancials: [FinancialDetailRecord] // 15 quarterly metrics
    let annualFinancials: [FinancialDetailRecord]    // 15 annual metrics

}
```

20+ nested types including `ChartPoint`, `InvestorFlow`, `AnalystOpinion`, `AnalystReport`, `DividendRecord`, `DividendYearGroup`, `FinancialRecord`, `FinancialDetailRecord`, and `OrderBook`.

```swift filename="PriceUpdate.swift"
struct PriceUpdate: Sendable, Decodable {
```
