<InfoBox type="info" title="What is data flow?">
Data flow is the journey your data takes from the internet to the screen. Think of it like a relay race: the **repository** fetches the data, passes it to the **view model** (which organizes it), and the **view** displays it to the user. Each runner has one job, and they pass the baton smoothly.
</InfoBox>

<InfoBox type="tip" title="How live updates work">
After the initial REST data loads, the WebSocket connection streams price updates in real time. The `HomeViewModel` stores these in a `livePrices` dictionary. SwiftUI views automatically re-render with smooth number animations whenever a price changes.
</InfoBox>

<StepGuide>
<Step number={1} title="View appears">
`HomeView` triggers `.task { loadHomeData(); startLiveUpdates() }` when it appears on screen.
</Step>
<Step number={2} title="ViewModel fetches data in parallel">
`HomeViewModel` fires off 6 repository calls at the same time using Swift's structured concurrency (`async let`).
</Step>
<Step number={3} title="REST data arrives">
Each repository method calls the backend REST API (or falls back to mock data). Results are stored in `@Observable` properties.
</Step>
<Step number={4} title="WebSocket connects">
`WebSocketManager` connects and subscribes to stock codes shown on the home screen.
</Step>
<Step number={5} title="Live prices stream in">
Trade ticks update the `livePrices` dictionary. SwiftUI views re-render with `.numericText()` animation for smooth price transitions.
</Step>
</StepGuide>

## Stock Detail Data Flow

The stock detail page combines REST data (for the full detail snapshot) with a per-stock WebSocket stream for live price overlay.

<InfoBox type="note" title="Live price overlay">
The ViewModel uses a simple pattern: `livePrice ?? stock.price`. When a WebSocket trade tick arrives, it overrides the REST price. If no live data is available yet, the REST price is shown instead.
</InfoBox>

<StepGuide>
<Step number={1} title="User taps a stock">
Navigation pushes `StockDetailPageView`, which triggers `.task { loadDetail(); startLiveUpdates() }`.
</Step>
<Step number={2} title="REST detail fetched">
`fetchStockDetail(id)` calls the backend, which returns the full stock snapshot including chart data, order book, and fundamentals.
</Step>
<Step number={3} title="Chart data downsampled">
`rebuildChartCache()` uses the LTTB algorithm to downsample chart data to 300 points for smooth rendering.
</Step>
<Step number={4} title="WebSocket subscribes">
`WebSocketManager` subscribes to the stock's code. An `AsyncStream` delivers each trade tick to the ViewModel.
</Step>
<Step number={5} title="Views render and update">
Child views (PriceHeader, ChartView, OrderBookView, DailyPriceView) display the data. Prices animate smoothly on each WebSocket tick.
</Step>
<Step number={6} title="Cleanup on leave">
`.onDisappear` calls `stopLiveUpdates()`, which cancels the async stream and unsubscribes after a 5-second grace period.
</Step>
</StepGuide>
