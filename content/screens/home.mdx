# Home Screen

The Home screen is the first thing you see when you open BARO. It is like a dashboard that shows you everything happening in the stock market right now. It has 7 sections that all load at the same time, each one getting its own data independently. If one section has a problem, the others keep working just fine.

    ### WebSocket Live Price Lifecycle

    <StepGuide>
      <Step number={1} title="Guard">
        Only domestic exchanges (KOSPI/KOSDAQ) qualify. Foreign exchanges call `stopLiveUpdates()`.
      </Step>
      <Step number={2} title="Connect">
        `WebSocketManager.shared.connect()`
      </Step>
      <Step number={3} title="Subscribe">
        `ws.subscribe(codes:)` with visible stock codes. Unsubscribe old codes first.
      </Step>
      <Step number={4} title="Poll">
        Background Task polls `latestPrices` every 300ms via `Task.sleep(for: .milliseconds(300))`.
      </Step>
      <Step number={5} title="Update">
        Compare `tradeTime`, update `livePrices[code]` if changed.
      </Step>
      <Step number={6} title="Re-subscribe">
        On filter/exchange change: `startLiveUpdates()` diffs old vs new codes.
      </Step>
      <Step number={7} title="Disconnect">
        `stopLiveUpdates()` cancels task, unsubscribes all codes, clears livePrices.
      </Step>
    </StepGuide>

    ### State Machine

    <InfoBox type="info" title="Section Loading Behavior">
      Each of the 7 sections loads independently. The ViewModel tracks 6 boolean loading flags. Sections show their own skeleton while loading and handle errors on their own.
    </InfoBox>

    | Section | Skeleton View | Error Behavior | Empty Behavior |
    |---|---|---|---|
    | H1 Header | None (static) | N/A | Always rendered |
    | H2 Banners | `BannerSkeletonView` | Silent fail, skeleton hides | Skeleton shown until load completes |
    | H3 Date | None (static) | N/A | Always rendered |
    | H4 Market Overview | `MarketOverviewRowSkeletonView` | Global error alert with retry | Skeleton shown when both empty + loading |
    | H5 Stock List | `StockListSkeletonView` | Silent fail, falls back to MockMarketRepository | Skeleton shown until stocks populated |
    | H6 Themes | None | Silent fail | Entire section hidden when empty |
    | H7 News | `NewsSectionSkeletonView` | Silent fail | "해당 필터의 뉴스가 없습니다" placeholder |

    ### Error Handling Strategy

    | Section | Strategy | Error Message (Korean) |
    |---|---|---|
    | Banners | Silent catch (print only) | N/A |
    | Indices | Alert with retry | "시장 지수를 불러올 수 없습니다. 네트워크 연결을 확인해주세요." |
    | FX Rates | Alert with retry (if no prior error) | "환율 데이터를 불러올 수 없습니다. 네트워크 연결을 확인해주세요." |
    | Stocks | Silent catch (print only) | N/A (API falls back to mock) |
    | News | Silent catch (print only) | N/A |
    | Themes | Silent catch (print only) | N/A |

    <Modal triggerLabel="View Data Loading Code" title="HomeViewModel.swift -- Parallel Data Loading" size="medium">

    ```swift
    // All 6 data sources load concurrently via async let
    func loadHomeData() async {
        async let loadBanners: Void = fetchBanners()
        async let loadIndices: Void = fetchIndices()
        async let loadRates: Void = fetchRates()
        async let loadStocks: Void = fetchStocks()
        async let loadNews: Void = fetchNews()
        async let loadThemes: Void = fetchThemes()

        _ = await (loadBanners, loadIndices, loadRates,
                   loadStocks, loadNews, loadThemes)
    }

    // Triggered on .task { } -- runs once on view appear
    .task {
        await viewModel.loadHomeData()
        viewModel.startLiveUpdates()
    }
    ```
