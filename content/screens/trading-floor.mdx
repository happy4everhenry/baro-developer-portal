## What Is the Trading Floor?

This is BARO's core trading screen. You see live bid/ask prices, tap in your order, and swipe to confirm. Every order goes through pre-trade risk checks to make sure everything is compliant with Korean financial regulations.

## FSC Compliance Requirements

<InfoBox type="warning" title="Financial Regulations (FSC)">
The Trading Floor must follow strict rules set by Korea's Financial Services Commission (FSC). These are not optional -- they are required by law for all Korean stock trading apps.
</InfoBox>

<StatGrid columns={3}>
  <StatCard value="2-step" label="Order confirmation required" />
  <StatCard value="1 sec" label="Cooling period between orders" />
  <StatCard value="30 min" label="Auto-logout on inactivity" />
</StatGrid>

- **Pre-trade risk checks:** Maximum order size validation, buying power verification
- **Order confirmation:** Mandatory 2-step confirmation for all orders
- **Audit trail:** All order actions logged with timestamps for regulatory compliance
- **Cooling period:** 1-second delay between consecutive orders
- **Session timeout:** Auto-logout after 30 minutes of inactivity

## Data Flow Architecture

<Tabs>
  <Tab label="Real-Time (WebSocket)" icon="⚡">
    Live price data streams through KIS WebSocket connections using venue-specific transaction IDs:

    | Data Type | KRX tr_id | NXT tr_id | Combined tr_id |
    |---|---|---|---|
    | Order Book | `H0STASP0` | `H0NXASP0` | `H0UNASP0` |
    | Trade Prices | `H0STCNT0` | `H0NXCNT0` | `H0UNCNT0` |
  </Tab>
  <Tab label="REST API (Planned)" icon="🌐">
    | Method | Path | What It Does |
    |---|---|---|
    | `POST` | `/api/orders` | Submit a new order |
    | `GET` | `/api/orders` | Get order history |
    | `DELETE` | `/api/orders/{id}` | Cancel a pending order |
    | `GET` | `/api/portfolio/positions` | Get current holdings |
  </Tab>
</Tabs>
