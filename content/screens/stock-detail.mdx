# Stock Detail Screen

This is the big, detailed page you see when you tap on any stock. It shows you everything about that stock: a live-updating price, an interactive chart you can pinch to zoom, and tabs full of information like order books, financials, dividends, and analyst opinions. Think of it as a stock's personal profile page.

    <InfoBox type="info" title="How is this screen built?">
      The screen splits into two layout zones. The top part (price header + chart + controls) lives in a regular `VStack` that is always in memory -- this prevents the chart from flickering when you switch tabs. The bottom part (content tabs) lives in a `LazyVStack` with pinned section headers so the tab bar sticks to the top while scrolling.

    <InfoBox type="tip" title="Why two VStacks?">
      The chart area is in a non-lazy `VStack` so it stays in memory even when you scroll down. This stops the chart from being deallocated and re-created (which would cause a flash) when you switch content tabs. The content below uses a `LazyVStack` for performance.
    </InfoBox>

    ### API Endpoint Mapping

    | Component | Repository Method | REST Endpoint | Response Model | Fallback |
    |---|---|---|---|---|
    | SD1, SD7-SD12 Detail | `marketRepo.fetchStockDetail(id:exchange:name:venue:)` | `GET /api/stocks/{code}/detail?venue={}` | `StockDetailInfo` | Print error, detailInfo remains nil |
    | SD2 Chart (initial) | `marketRepo.fetchChartData(code:exchange:period:venue:)` | `GET /api/stocks/{code}/chart?period=1d&venue={}` | `[ChartPoint]` | Local fallback: filter `detailInfo.chartData` |
    | SD2 Chart (period) | `marketRepo.fetchChartData(code:exchange:period:venue:)` | `GET /api/stocks/{code}/chart?period={}&venue={}` | `[ChartPoint]` | Local fallback + cancellable |
    | SD1 Live Price | `WebSocketManager.shared` | WebSocket subscribe(codes:, venue:) | `PriceUpdate` | REST snapshot only (domestic only) |
    | SD13 Order Book | Included in `fetchStockDetail` | orderBook field | `OrderBook` | ProgressView spinner |
    | SD14 Daily Prices | Included in `fetchStockDetail` | chartData field | `[ChartPoint]` | ProgressView spinner |

    ### Chart Rendering Pipeline

    <StepGuide>
      <Step number={1} title="Fetch">
        On period change or initial load, call `fetchChartData()` from backend. Output: `periodChartData: [ChartPoint]` (raw API response).
      </Step>
      <Step number={2} title="Fallback">
        If API is empty or errors, run `useLocalChartFallback()`: calendar-based filter of cached data.
      </Step>
      <Step number={3} title="Cache Rebuild">
        `rebuildChartCache()`: resetZoom, compute min/max/range, detect trend direction. Output: `chartPoints`, `chartMinPrice`, `chartMaxPrice`, `chartIsPositive`.
      </Step>
      <Step number={4} title="Downsample">
        LTTB (Largest-Triangle-Three-Buckets) when count > 300. Output: `displayChartPoints` (max 300 pts).
      </Step>
      <Step number={5} title="Zoom Slice">
        Per render frame (when zoomed): `visibleDisplayPoints(zoom, panOffset)` + `zoomedPriceStats()`.
      </Step>
      <Step number={6} title="Render">
        Canvas rendering: multi-color segments, gradient fill, candlestick/OHLC/step. 200pt chart with annotations and price badge.
      </Step>
    </StepGuide>

    ### State Machine

    <InfoBox type="info" title="Two Independent Loading States">
      The screen manages `isLoading` for detail and `isLoadingChart` for chart data. Both resolve in parallel. WebSocket streaming starts only after the initial REST load finishes.
    </InfoBox>

    **Primary Flow:** onAppear -> loadDetail() (async let parallel) -> Detail + Chart Loaded -> rebuildChartCache() -> startLiveUpdates()

    **Chart Period Change:** Loaded -> selectedPeriod didSet (cancel prior) -> loadChartForPeriod() -> rebuildChartCache() + resetZoom()

    **Venue Change:** Loaded -> selectedVenue didSet (cancel prior) -> switchLiveUpdatesVenue() + loadDetail() -> New venue data + WS re-subscribed

    **Chart Fallback:** fetchChartData() -> API Empty/Error -> useLocalChartFallback() + rebuildChartCache()

    | Section | Skeleton / Placeholder | Error Behavior | Empty Behavior |
    |---|---|---|---|
    | SD1 Price Header | None (uses stock.price immediately) | N/A | Always rendered |
    | SD2 Chart | Empty rectangle (klBackground, 200pt) | Falls back to `useLocalChartFallback()` | Blank chart area |
    | SD7 Info Grid | `SkeletonBlock` grid with `.shimmer()` | Skeleton remains visible | "항목을 추가해 주세요" |
    | SD8-SD12 Sections | Part of info grid skeleton | Hidden until detailInfo loads | Rendered with available data |
    | SD13 Order Book | Centered `ProgressView()` spinner | Spinner remains visible | Spinner until detailInfo loads |
    | SD14 Daily Prices | Centered `ProgressView()` spinner | Spinner remains visible | Spinner until detailInfo loads |

    <Modal triggerLabel="View Layout Architecture Code" title="Split Layout Pattern" size="large">

    ```swift
    ScrollViewReader { scrollProxy in
        ScrollView {
            VStack(spacing: 0) {                       // NON-LAZY: always allocated
                StockPriceHeaderView(...)              // [SD1, SD15]
                StockChartView(...)                    // [SD2] height: 200pt
                HStack { IndicatorPills; zoomControls } // [SD4, SD3]
                StockDetailPeriodTabsView(...)         // [SD5]
                Divider()
            }
            .background(Color.klSurface)

            LazyVStack(pinnedViews: [.sectionHeaders]) { // LAZY: content tabs
                Section {
                    contentForSelectedTab               // [SD7-SD14]
                        .frame(minHeight: screenHeight)  // ensures scrollability
                } header: {
                    KLSegmentedControl(...)             // [SD6] pinned
                        .id("contentTabs")              // scroll target
                }
            }
        }
        .onChange(of: selectedContentTab) {
            withAnimation(nil) {
                scrollProxy.scrollTo("contentTabs", anchor: .top)
            }
        }
    }
    ```
