# 호가 및 매매

<PageMeta status="live" layer="shared" />

주식의 실시간 거래가 이루어지는 **호가창(Order Book)**의 구조와 매매 개념을 설명합니다. 앱에서는 `StockOrderBookView`가 이 화면을 구현합니다.

---

## 호가창(Order Book)이란?

호가창은 **"이 가격에 사고 싶다" / "이 가격에 팔고 싶다"** 라는 주문들을 모아 보여주는 화면입니다. 실시간으로 업데이트됩니다.

### 구조

```
  ── 매도 호가 (Ask) ──   팔려는 사람들
  72,800원   1,500주    ← 가장 비싼 매도 호가 (10호가)
  72,700원   2,300주
  72,600원   1,800주
  72,500원   3,200주    ← 가장 싼 매도 호가 (1호가)
  ─────────────────────
  ★ 현재가 72,400원 ★   ← 마지막 체결 가격
  ─────────────────────
  72,400원   2,800주    ← 가장 비싼 매수 호가 (1호가)
  72,300원   1,900주
  72,200원   2,100주
  72,100원   1,200주    ← 가장 싼 매수 호가 (10호가)
  ── 매수 호가 (Bid) ──   사려는 사람들
```

### 핵심 용어

| 용어 | 영문 | 설명 |
|------|------|------|
| **매도 호가 (Ask)** | Ask Price | 이 가격에 **팔겠다**는 주문. 호가창 위쪽 (빨간색 영역) |
| **매수 호가 (Bid)** | Bid Price | 이 가격에 **사겠다**는 주문. 호가창 아래쪽 (파란색 영역) |
| **잔량** | Quantity | 해당 가격에 대기 중인 주식 수량 |
| **체결** | Execution/Fill | 매수와 매도 가격이 만나서 실제 거래가 성사되는 것 |
| **스프레드 (Spread)** | Bid-Ask Spread | 최우선 매도가와 최우선 매수가의 차이. 작을수록 유동성이 좋음 |

### 앱 구현

```swift
// StockOrderBookView.swift — 호가창 구현
struct StockOrderBookView: View {
    let orderBook: OrderBook    // 호가 데이터
    let prevClose: Int          // 전일 종가 (색상 판단 기준)

    // maxQuantity: 가로 막대 길이를 계산하기 위한 최대 잔량
    private var maxQuantity: Int  // 모든 호가 중 가장 큰 잔량
}
```

```swift
// OrderBook 모델 구조
struct OrderBook {
    let asks: [OrderBookEntry]  // 매도 호가 10단계 (비싼 순)
    let bids: [OrderBookEntry]  // 매수 호가 10단계 (비싼 순)
}

struct OrderBookEntry {
    let price: Int       // 호가 가격
    let quantity: Int     // 해당 가격의 잔량
}
```

### 가로 막대 (Volume Bar)

호가창에서 각 가격 옆의 **가로 막대**는 해당 가격의 잔량을 시각적으로 나타냅니다.

```
막대 길이 = 해당 호가 잔량 / 전체 호가 중 최대 잔량
```

- 막대가 길다 = 그 가격에 대기 주문이 많다
- 매도 쪽 막대가 전체적으로 크면 → 매도 압력이 강함 (가격 하락 가능성)
- 매수 쪽 막대가 전체적으로 크면 → 매수 압력이 강함 (가격 상승 가능성)

### 총잔량 요약 (Summary Bar)

호가창 하단의 **요약 막대**는 총 매도 잔량과 총 매수 잔량의 비율을 보여줍니다.

```swift
// StockOrderBookView.swift — summaryBar 구현 (line 205)
총 매도 잔량: 15,200주  |  총 매수 잔량: 12,100주
[████████████████░░░░░░░░░░░░]
     매도 55.7%      매수 44.3%
```

## 가격 색상 규칙

호가창에서 가격 색상은 **전일종가 대비** 결정됩니다:

| 조건 | 색상 | 의미 |
|------|------|------|
| 가격 > 전일종가 | 🔴 빨간색 | 전일보다 비쌈 (상승) |
| 가격 < 전일종가 | 🔵 파란색 | 전일보다 쌈 (하락) |
| 가격 = 전일종가 | ⚫ 기본색 | 변동 없음 |

```swift
// StockOrderBookView.swift — colorForPrice (line 247)
private func colorForPrice(_ price: Int) -> Color {
    if price > prevClose { return Color.klPriceUp }     // 빨간색
    if price < prevClose { return Color.klPriceDown }   // 파란색
    return Color.klText                                  // 기본색
}
```

## 매매 주문 유형 (Phase 2 계획)

`TradingFloorView`에서 구현 예정인 주문 유형입니다:

| 주문 유형 | 설명 |
|-----------|------|
| **시장가 주문 (Market Order)** | 현재 시장 가격에 즉시 체결. 가격보다 속도가 중요할 때 사용 |
| **지정가 주문 (Limit Order)** | 원하는 가격을 지정하여 주문. 해당 가격에 도달하면 체결 |
| **스톱 주문 (Stop Order)** | 가격이 특정 수준에 도달하면 자동으로 주문 실행 |

## 일별시세 (Daily Price Table)

`StockDailyPriceView`에서 보여주는 과거 거래일별 가격 데이터입니다.

| 컬럼 | 설명 |
|------|------|
| 일자 | 거래일 날짜 |
| 종가 | 해당일의 마감 가격 |
| 대비 | 전일 대비 변동액 |
| 등락률 | 전일 대비 변동률 (%) |
| 거래량 | 해당일 총 거래 수량 |

> **개발자 참고:** 컬럼 정렬이 가능합니다. `toggleSort()` 함수가 `sortColumn`과 `sortAscending` 상태를 관리합니다.

## 실시간 데이터 (WebSocket)

호가 데이터는 **WebSocket**을 통해 실시간으로 업데이트됩니다.

| tr_id | 설명 |
|-------|------|
| `H0STASP0` | KRX 실시간 호가 |
| `H0NXASP0` | NXT 실시간 호가 |
| `H0UNASP0` | 통합 실시간 호가 |
| `H0STCNT0` | KRX 실시간 체결 |
| `H0NXCNT0` | NXT 실시간 체결 |

```swift
// WebSocketManager.swift — 종목별 가격 구독
func subscribe(stockCode: String) { ... }
func unsubscribe(stockCode: String) { ... }
```

> **개발자 참고:** WebSocket 연결이 끊어지면 exponential backoff (최대 60초)로 자동 재연결합니다. UI 업데이트는 300ms 간격으로 throttle됩니다.

## FSC 규정 준수 (Phase 2)

한국 금융위원회(FSC) 규정에 따라 매매 시 다음 사항이 필수입니다:

| 규정 | 설명 |
|------|------|
| 선주문 위험 점검 | 주문 전 최대 주문 규모, 구매력 검증 |
| 2단계 확인 | 모든 주문에 대해 확인 화면 표시 필수 |
| 감사 추적 | 모든 주문 행위를 타임스탬프와 함께 로깅 |
| 쿨링 기간 | 연속 주문 사이 1초 지연 |
| 세션 타임아웃 | 30분 비활동 시 자동 로그아웃 |
