</InfoBox>

## Setup Steps

<Tabs>
  <Tab label="Backend Setup" icon="🐍">

<StepGuide>
  <Step number={1} title="Go to the project folder" icon="📂">

Open your terminal and navigate to the backend folder:

```bash
cd ~/Desktop/BARO/backend
```

  </Step>

  <Step number={2} title="Create a virtual environment" icon="📦">

A virtual environment keeps BARO's Python packages separate from everything else on your Mac. Think of it like a clean room just for this project.

```bash
python3 -m venv venv
source venv/bin/activate
```

<InfoBox type="info" title="How do I know it worked?">
You should see `(venv)` at the beginning of your terminal prompt. That means you're inside the virtual environment!
</InfoBox>

  </Step>

  <Step number={3} title="Install dependencies" icon="⬇️">

This downloads all the Python libraries BARO needs (FastAPI, WebSocket tools, the KIS API client, and more):

```bash
pip install -r requirements.txt
```

  </Step>

  <Step number={4} title="Set up your API keys" icon="🔑">

Copy the example environment file and fill in your real credentials:

```bash
cp .env.example .env
```

Then open `.env` in any editor and fill in these three values:

```bash
KIS_APP_KEY=your_app_key
KIS_APP_SECRET=your_app_secret
KIS_ACCOUNT_NO=your_account_number
```

<InfoBox type="danger" title="Keep your keys secret!">
Never commit your `.env` file to git. It contains private API credentials.
</InfoBox>

  </Step>

  <Step number={5} title="Start the server" icon="🚀">

This launches the backend! The `--reload` flag means it restarts automatically when you change code.

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

<InfoBox type="tip" title="Is it running?">
Open your browser and go to `http://localhost:8000/health`. You should see `{"status": "ok"}`.
</InfoBox>

  </Step>
</StepGuide>

  </Tab>

  <Tab label="iOS Setup" icon="📱">

<StepGuide>
  <Step number={1} title="Open the Xcode project" icon="💻">

Navigate to the iOS folder and open the project in Xcode:

```bash
cd ~/Desktop/BARO/ios/BARO
open BARO.xcodeproj
```

Or, if you prefer building from the command line:

```bash
xcodebuild -project BARO.xcodeproj \
  -scheme BARO \
  -destination 'platform=iOS Simulator,name=iPhone 16 Pro' \
  build
```

  </Step>

  <Step number={2} title="Run in the Simulator" icon="▶️">

Press **Cmd + R** in Xcode (or click the Play button). The app will build and launch in the iPhone Simulator.

<InfoBox type="warning" title="Simulator vs. Physical Device">
The app connects to different addresses depending on where it runs:

- **Simulator:** `127.0.0.1:8000` (your Mac's localhost)
- **Physical device:** Your Mac's local Wi-Fi IP on port `8000`

Replace `192.168.68.57` with your machine's local IP address (find via `ifconfig` on macOS or `ipconfig getifaddr en0`). You can change this in `WebSocketManager.swift`.
</InfoBox>

  </Step>

  <Step number={3} title="Make sure the backend is running!" icon="🔗">

The iOS app needs the Python backend to show real data. If you haven't started it yet, switch to the **Backend Setup** tab and follow those steps first.

  </Step>
</StepGuide>

  </Tab>
</Tabs>

## Key Architecture Decisions

| Decision | Choice | Why? |
|----------|--------|------|
| UI Framework | SwiftUI + @Observable | Modern reactive UI with minimal boilerplate |
| Real-time Data | Native WebSocket | Low latency, venue-aware subscriptions |
| Backend | FastAPI (Python) | Async WebSocket support, rapid prototyping |
| Data Source | KIS (한국투자증권) API | Official Korean market data with KRX + NXT venues |
| Charts | WKWebView + Lightweight Charts | TradingView-quality charts via web bridge |
| Multi-venue | KRX / NXT / Combined | Support for both traditional KRX and new NXT exchange |
