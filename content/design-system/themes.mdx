## Light Theme <StatusBadge status="live">IMPLEMENTED</StatusBadge>

The default and currently shipping theme. Clean white surfaces on a light gray canvas with subtle borders and minimal shadows. Follows iOS HIG light appearance conventions.

- Full color token coverage (37 tokens)
- All components styled and tested
- Chart themes aligned with color system
- Navigation and tab bar themed
- Production ready

### What It Looks Like

- Background: `klBackground` (#F2F3F5)
- Card surface: `klSurface` (#FFFFFF) with 1px `klBorder` (#F0F0F0) and Level 1 shadow
- Text: `klText` (#1A1A1A) for names, `klTextTertiary` (#AAAAAA) for ticker codes
- Prices: `klText` (#1A1A1A), changes: `klPriceUp` (#E04848) or `klPriceDown` (#3478F6)

### Color Contrast & Accessibility

All combinations must meet WCAG 2.1 AA contrast ratios.

| Pairing | Light Contrast | Dark Contrast (Est.) | WCAG AA |
|---------|---------------|---------------------|---------|
| `klText` on `klSurface` | #1A1A1A on #FFF -- 17.4:1 | #F0F0F0 on #0A0A0F -- 16.8:1 | <StatusBadge status="live">Pass</StatusBadge> |
| `klTextSecondary` on `klSurface` | #555 on #FFF -- 7.5:1 | #A0A0A0 on #0A0A0F -- 8.2:1 | <StatusBadge status="live">Pass</StatusBadge> |
| `klTextTertiary` on `klSurface` | #AAA on #FFF -- 2.3:1 | #666 on #0A0A0F -- 3.1:1 | <StatusBadge status="wip">Decorative</StatusBadge> |
| `klPriceUp` on `klSurface` | #E04848 on #FFF -- 4.0:1 | #E04848 on #0A0A0F -- 4.4:1 | <StatusBadge status="live">Pass (large)</StatusBadge> |
| `klPriceDown` on `klSurface` | #3478F6 on #FFF -- 4.6:1 | #3478F6 on #0A0A0F -- 4.9:1 | <StatusBadge status="live">Pass (large)</StatusBadge> |

<InfoBox type="warning" title="Tertiary Text Note">
`klTextTertiary` and `klTextQuaternary` do not meet WCAG AA contrast minimums. These tokens are for decorative or supplementary metadata only (timestamps, ticker codes) that always appear alongside primary-contrast labels. Never use them for essential information.
</InfoBox>

## Dark Theme <StatusBadge status="planned">PLANNED</StatusBadge>

The planned dark appearance mode uses the Liquid Glass design system with glass-morphism effects, mesh gradients, and specular highlights.

<ProgressBar value={15} label="Dark Theme Completion" color="blue" />

### What Is Planned

- Token remapping required (37 tokens)
- Glass-morphism surface effects
- Animated mesh gradient backgrounds
- Specular overlay system
- Estimated: Phase 2

### What It Will Look Like

- Background: `klBackground` (#0A0A0F) with mesh gradient orbs (jade, violet)
- Card surface: glass `white/0.06` with 1px `white/0.08` border and backdrop-filter blur
- Text: `klText` (#F0F0F0) for names, `klTextTertiary` (#666666) for ticker codes
- Prices: unchanged -- `klPriceUp` (#E04848) and `klPriceDown` (#3478F6) are invariant

### Liquid Glass Surface Tiers

| Tier | Background | Blur |
|------|-----------|------|
| `--glass` (Level 1) | `rgba(255,255,255,0.04)` | `backdrop-filter: blur(20px)` |
| `--glass-2` (Level 2) | `rgba(255,255,255,0.06)` | `backdrop-filter: blur(24px)` |
| `--glass-3` (Level 3) | `rgba(255,255,255,0.09)` | `backdrop-filter: blur(30px)` |

### Mesh Gradient Orbs

Animated gradient orbs provide ambient background color in dark mode: jade, amber, violet, and red radial gradients positioned across the canvas.

### Specular Highlights

Glass surfaces include a specular gradient overlay that simulates light reflecting off glass edges -- a subtle top-to-bottom gradient creating the illusion of a light source above the UI.

```css filename="Specular Gradient"
--lq-specular: linear-gradient(
  180deg,
  rgba(255, 255, 255, 0.15) 0%,
  rgba(255, 255, 255, 0.02) 40%,
  transparent 100%
);
```

### Theme Switching Behavior

| Behavior | Specification |
|----------|---------------|
| **System Follow** | Default. Matches iOS system appearance via `@Environment(\.colorScheme)`. |
| **Manual Override** | Force light or dark via Settings > Appearance. Stored in `UserDefaults`. |
| **Transition** | Cross-fade with 0.3s ease-in-out animation. No jarring snap. |
| **Chart Adaptation** | Chart backgrounds, grid lines, tooltips, and axis labels adapt to active theme. |
| **Glass Effects** | Backdrop blur and specular highlights active only in dark mode. |
| **Persistence** | Theme preference persisted via `@AppStorage("kl_theme")`. |

<AccordionItem title="Swift Theme Architecture" icon="code" defaultOpen={false}>

```swift filename="Planned Theme Architecture"
// Future: KLINETheme.swift
enum KLINETheme {
    case light
    case dark

    var background: Color {
        switch self {
        case .light: return Color(hex: "F2F3F5")
        case .dark:  return Color(hex: "0A0A0F")
        }
    }

    var surface: Color {
        switch self {
        case .light: return .white
        case .dark:  return Color.white.opacity(0.06)
        }
    }

    var text: Color {
        switch self {
        case .light: return Color(hex: "1A1A1A")
        case .dark:  return Color(hex: "F0F0F0")
        }
    }

    // Price semantics remain constant across themes
    var priceUp: Color { Color(hex: "E04848") }
    var priceDown: Color { Color(hex: "3478F6") }
}

// Usage in views
struct StockRow: View {
    @Environment(\.colorScheme) var colorScheme

    var theme: KLINETheme {
        colorScheme == .dark ? .dark : .light
    }

    var body: some View {
        HStack {
            Text(stock.name)
                .foregroundColor(theme.text)
        }
        .background(theme.surface)
    }
}
```

## Theme Guidelines

  <FeatureCard icon="check" title="Do" description="Always use semantic tokens (e.g. klText) not hardcoded hex. Test all views in both simulators. Maintain WCAG 2.1 AA contrast.   <FeatureCard icon="x" title="Don't" description="Don't hardcode light-mode hex values. Don't modify price semantics for dark theme. Don't assume white backgrounds.
