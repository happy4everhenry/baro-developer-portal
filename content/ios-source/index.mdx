## Purpose

This documentation is designed for **future maintainers** of the BARO iOS application. It provides:

- **Complete file inventory** — Every Swift source file with its purpose, location, and responsibilities
- **Architecture deep-dive** — How the MVVM + Repository pattern is implemented in practice
- **Module breakdowns** — Feature-by-feature documentation of views, view models, and components
- **Infrastructure reference** — Network layer, WebSocket, design system internals
- **Coding conventions** — Naming patterns, formatting rules, and idioms used throughout the codebase
- **Maintenance playbook** — Step-by-step guides for common tasks like adding new screens, models, or API endpoints

## Tech Stack Summary

| Technology | Version | Purpose |
|-----------|---------|---------|
| Swift | 5.9+ | Primary language |
| SwiftUI | iOS 16+ | UI framework |
| @Observable | Observation framework | State management (replaces ObservableObject) |
| async/await | Swift Concurrency | Asynchronous operations |
| AsyncStream | Swift Concurrency | WebSocket event streaming |
| URLSession | Foundation | HTTP networking |
| WebSocket | URLSessionWebSocketTask | Real-time price streaming |
