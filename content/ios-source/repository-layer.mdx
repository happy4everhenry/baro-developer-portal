## Protocol Definitions

### MarketRepositoryProtocol.swift

**Location:** `Repositories/Protocols/MarketRepositoryProtocol.swift`

The primary data access protocol — all 9 methods are `async throws`:

| Method | Returns | Description |
|--------|---------|-------------|
| `fetchIndices()` | `[MarketIndex]` | Top market indices (KOSPI, KOSDAQ, etc.) |
| `fetchAllIndices()` | `[MarketIndex]` | All available market indices |
| `fetchStocks(filter:exchange:)` | `[Stock]` | Filtered stock list (top 5) |
| `fetchDetailStocks(filter:exchange:)` | `[Stock]` | Filtered stock list (extended) |
| `fetchCurrencyRates()` | `[CurrencyRate]` | Foreign exchange rates |
| `fetchSearchStocks(domestic:)` | `[Stock]` | All stocks for search pool |
| `fetchStockDetail(id:exchange:name:venue:)` | `StockDetailInfo` | Full stock detail with all sections |
| `fetchChartData(code:exchange:period:venue:)` | `[ChartPoint]` | Historical chart data |
| `fetchMarketThemes()` | `[MarketTheme]` | Market sector themes |

**Default Parameter Extension:**

```swift
extension MarketRepositoryProtocol {
    func fetchStockDetail(id: String, exchange: StockExchange,
                          name: String) async throws -> StockDetailInfo {
        try await fetchStockDetail(id: id, exchange: exchange,
                                   name: name, venue: .combined)
    }
}
```

## API Implementation

### APIMarketRepository.swift

**Location:** `Repositories/API/APIMarketRepository.swift` (~360 lines)

Live API implementation that calls the FastAPI backend via `APIClient`.

**REST Endpoints Used:**

| Method | Endpoint | Source |
|--------|----------|--------|
| `fetchIndices()` | `GET /api/market/indices/overview` | Yahoo Finance via backend |
| `fetchAllIndices()` | `GET /api/market/indices` | Yahoo Finance via backend |
| `fetchCurrencyRates()` | `GET /api/market/currency-rates` | Yahoo Finance via backend |
| `fetchStocks()` | `GET /api/stocks?exchange=&filter=&limit=5` | KIS Open API |
| `fetchDetailStocks()` | `GET /api/stocks?exchange=&filter=&detail=true` | KIS Open API |
| `fetchStockDetail()` | `GET /api/stocks/{id}/detail?exchange=&name=&venue=` | KIS Open API |
| `fetchChartData()` | `GET /api/stocks/{code}/chart?exchange=&period=&venue=` | KIS Open API |
| `fetchSearchStocks()` | Falls back to mock | No search API available |
| `fetchMarketThemes()` | Falls back to mock | No themes API available |

**Fallback Strategy:**

Every method wraps the network call in `do/catch` with automatic fallback to `MockMarketRepository`:

```swift
func fetchIndices() async throws -> [MarketIndex] {
    do {
        return try await apiClient.get("/api/market/indices/overview")
    } catch {
        print("[API] Indices failed, using mock: \(error)")
        return try await MockMarketRepository().fetchIndices()
    }
}
```

> **Note for maintainers:** This fallback pattern means the app never crashes due to API failures — it gracefully degrades to mock data. However, this can mask API issues during development. Check console logs for `[API] ... failed, using mock:` messages.

**MockDataFiller (nested struct):**

For `fetchStockDetail()`, the KIS API only provides price and chart data. Extended sections are filled with deterministic mock data:

| Section | Data Source |
|---------|------------|
| Price, Chart, Order Book | KIS Open API (real data) |
| Analyst Opinions | MockDataFiller (15 opinions) |
| Research Reports | MockDataFiller (12 reports) |
| Dividends | MockDataFiller (4 years quarterly) |
| Financials | MockDataFiller (15 metrics) |
| Investor Flow | MockDataFiller (15 days) |

## Dependency Injection Container

### DependencyContainer.swift

**Location:** `App/DependencyContainer.swift`

```swift
@Observable
final class DependencyContainer {
    let marketRepository: any MarketRepositoryProtocol
    let bannerRepository: any BannerRepositoryProtocol
    let newsRepository: any NewsRepositoryProtocol
    let eventRepository: any EventRepositoryProtocol
    let authManager: AuthManager

    init(useMocks: Bool = false) {
        authManager = AuthManager(tokenProvider: { KeychainHelper.read(key: "com.baro.accessToken") })
        if useMocks {
            marketRepository = MockMarketRepository()
            bannerRepository = MockBannerRepository()
            newsRepository = MockNewsRepository()
            eventRepository = MockEventRepository()
        } else {
            marketRepository = APIMarketRepository()
            bannerRepository = MockBannerRepository()   // Still mock (no API yet)
            newsRepository = MockNewsRepository()        // Still mock (no API yet)
            eventRepository = MockEventRepository()      // Still mock (no API yet)
        }
    }
}
```

**Current State:**
- `useMocks: true` in `BAROApp.swift` — all data from mocks
- `useMocks: false` — market data from API, banners/news still from mocks (no API endpoints exist yet)

**How to switch to live API:**

1. Open `BAROApp.swift`
2. Change `DependencyContainer(useMocks: true)` → `DependencyContainer(useMocks: false)`
3. Ensure the FastAPI backend is running at the correct IP address
4. Update `APIClient.swift` device IP if on physical device

**Adding a new repository:**

1. Create protocol in `Repositories/Protocols/`
2. Create mock implementation in `Repositories/Mock/`
3. Create API implementation in `Repositories/API/`
4. Add property to `DependencyContainer`
5. Initialize in both `if useMocks` branches
