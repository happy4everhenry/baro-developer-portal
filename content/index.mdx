## How the App is Built

<InfoBox type="tip" title="Think of it like building blocks!">
  BARO is built in **layers**, like stacking LEGO blocks. Each layer has a specific job. The screen you see talks to a "brain" (ViewModel), which asks a "helper" (Repository) to get data from the internet (API).
</InfoBox>

## Tech Stack at a Glance

<Tabs>
  <Tab label="📱 iOS App" icon="📱">
    | Technology | What it does |
    |---|---|
    | **SwiftUI** | Builds the visual interface — buttons, lists, charts |
    | **@Observable** | Automatically updates the screen when data changes |
    | **async/await** | Fetches data from the internet without freezing the app |
    | **Canvas** | Draws high-performance stock charts |
    | **Combine** | Handles real-time price streams via WebSocket |
  </Tab>
  <Tab label="🖥️ Backend" icon="🖥️">
    | Technology | What it does |
    |---|---|
    | **Python + FastAPI** | Runs the server that the app talks to |
    | **WebSocket** | Sends live stock prices instantly |
    | **KIS API** | Connects to Korea Investment & Securities for real data |
    | **REST** | Standard request/response API pattern |
  </Tab>
  <Tab label="🏗️ Patterns" icon="🏗️">
    | Pattern | What it means |
    |---|---|
    | **MVVM** | Separates what you see (View) from the logic (ViewModel) and data (Model) |
    | **Repository Pattern** | One central place to get data — can switch between fake and real data easily |
    | **Dependency Injection** | Lets you swap parts in and out, like changing batteries |
    | **Protocol-based** | Defines "contracts" so different parts know how to talk to each other |
  </Tab>
</Tabs>
