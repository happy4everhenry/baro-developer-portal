## How the App is Built

<InfoBox type="tip" title="Think of it like building blocks!">
  BARO is built in **three layers** — a SwiftUI iOS app, a Python FastAPI backend, and an AI platform. Each layer has a specific job, and they all work together to deliver real-time market data with intelligent insights.
</InfoBox>

## Tech Stack at a Glance

<Tabs>
  <Tab label="📱 iOS App" icon="📱">
    | Technology | What it does |
    |---|---|
    | **SwiftUI** | Builds the visual interface — buttons, lists, charts |
    | **@Observable** | Automatically updates the screen when data changes |
    | **async/await** | Fetches data from the internet without freezing the app |
    | **Canvas** | Draws high-performance stock charts |
    | **Combine** | Handles real-time price streams via WebSocket |
  </Tab>
  <Tab label="🖥️ Backend" icon="🖥️">
    | Technology | What it does |
    |---|---|
    | **Python + FastAPI** | Async server with 25+ REST endpoints and WebSocket relay |
    | **KIS Open API** | Korea Investment & Securities — real-time Korean stock data |
    | **Yahoo Finance** | Global market indices and FX rates |
    | **SQLite + aiosqlite** | User authentication database (async) |
    | **PyJWT + bcrypt** | JWT token management and password hashing |
    | **WebSocket relay** | Translates KIS binary feed into clean JSON for iOS |
    | **httpx** | Async HTTP client for KIS API calls with rate limiting |
  </Tab>
  <Tab label="🤖 AI Platform" icon="🤖">
    | Technology | What it does |
    |---|---|
    | **Claude (Anthropic)** | Primary LLM for analysis, chatbot, and report generation |
    | **LangGraph** | Orchestrates 12 AI agents with typed state graphs |
    | **BGE-M3 + pgvector** | Embedding model + vector database for semantic search |
    | **KoBERT / FinBERT-KO** | Korean financial sentiment analysis |
    | **6 MCP Servers** | 38 standardized tools for trading, compliance, and retrieval |
    | **4-Layer Firewall** | Input, output, inter-agent, and training pipeline safety |
    | **EXAONE 3.5 7.8B** | Local Korean LLM for cost-efficient routing |
  </Tab>
  <Tab label="🏗️ Patterns" icon="🏗️">
    | Pattern | What it means |
    |---|---|
    | **MVVM** | Separates what you see (View) from the logic (ViewModel) and data (Model) |
    | **Repository Pattern** | One central place to get data — can switch between fake and real data easily |
    | **Dependency Injection** | Lets you swap parts in and out, like changing batteries |
    | **Protocol-based** | Defines "contracts" so different parts know how to talk to each other |
    | **CAG + RAG** | Context-augmented and retrieval-augmented generation for AI knowledge |
    | **Human-in-the-Loop** | AI stages actions, humans confirm — no autonomous money movement |
  </Tab>
</Tabs>
