## Main Components

These are the big building blocks that make up the stock detail page from top to bottom.

<Accordion>
<AccordionItem title="StockDetailPageView (Root Container)" icon="📱" badge="Root" defaultOpen={true}>

The root container for the stock detail screen. Presented as a `fullScreenCover` from stock list rows. It manages tab-based content switching, real-time WebSocket price updates, and chart data caching.

### Architecture

```
StockDetailPageView
  ├── KLDetailNavBar (title: stock name, trailing: home + star buttons)
  ├── StockPriceHeaderView (live price, change, market/NXT venue tabs)
  ├── StockChartView (interactive chart with zoom/scrub)
  ├── IndicatorPillsView (active overlay/sub-indicator badges)
  ├── StockDetailPeriodTabsView (1D/1W/1M/3M/1Y/5Y + settings gear)
  ├── KLSegmentedControl (section tabs)
  └── StockDetailSectionsView (content panes)
      ├── Order Book (StockOrderBookView)
      ├── Daily Price (StockDailyPriceView)
      ├── Stock Info (StockInfoGridView)
      ├── Financials (FinancialDetailView)
      ├── Dividends (DividendDetailView)
      ├── Investor Flow (InvestorFlowDetailView)
      ├── Analyst Opinions (AnalystOpinionDetailView)
      └── Reports (ReportDetailView)
```

### MarketVenue <KoreanLabel>시장 구분</KoreanLabel>

| Case | Korean | Description |
|------|--------|-------------|
| `.krx` | 정규장 | Korea Exchange regular session |
| `.nxt` | NXT | Korea New Exchange (alternative venue) |
| `.combined` | 통합 | Combined KRX + NXT volume/prices |

```swift filename="StockDetailPageViewModel.swift"
// Live price overlay: prefer WebSocket data
var displayPrice: Int { livePrice?.price ?? detail?.stock.price ?? stock.price }
var displayChange: Int { livePrice?.change ?? detail?.stock.change ?? stock.change }

// LTTB downsampling for chart performance
func rebuildChartCache() {
    let raw = detail?.chartData ?? []
    displayPoints = raw.count > 300 ? lttbDownsample(raw, targetCount: 300) : raw
}
```

<AccordionItem title="StockPriceHeaderView" icon="💰" badge="Price">

Large price display with current price, absolute change, percentage change, and venue selector tabs (정규장/NXT/통합). The price numbers animate digit-by-digit when they update!

| Element | Spec |
|---------|------|
| Price | `.system(size: 32, weight: .bold)`, `klText` color, `.numericText()` transition |
| Change | `PriceChangeLabel` with `.system(size: 14, weight: .semibold)` |
| Venue tabs | 3 capsule pills (정규장 / NXT / 통합), active: `klText` on `klSurface`, inactive: `klTextTertiary` |
| NXT badge | Green "NXT" label, shown only for NXT-eligible stocks (`stock.isNXT`) |
| Fractional badge | "소수점" label, shown for fractional-share eligible stocks (`stock.isFractional`) |

```swift filename="StockPriceHeaderView.swift"
Text(displayPrice.formattedWon)
    .font(.system(size: 32, weight: .bold))
    .foregroundStyle(Color.klText)
    .contentTransition(.numericText())
    .animation(.easeInOut(duration: 0.3), value: displayPrice)
```

</AccordionItem>

<AccordionItem title="StockOrderBookView" icon="📊" badge="Order Book">

10-level order book with ask (sell) prices on top and bid (buy) prices on bottom. Each level shows price, change from previous close, quantity, and a proportional bar.

<InfoBox type="info" title="What is an order book?">
An order book is like a scoreboard showing how many people want to buy vs sell at each price. The top half (blue) shows sellers, and the bottom half (red) shows buyers. The wider the bar, the more people are waiting at that price!
</InfoBox>

| Element | Spec |
|---------|------|
| Ask rows | 10 levels, blue text (`klPriceDown`), right-aligned quantity bars |
| Bid rows | 10 levels, red text (`klPriceUp`), right-aligned quantity bars |
| Center divider | Current price with change indicator |
| Bar width | Proportional to `quantity / maxQuantity`, max 60% of row width |
| Price format | `.formattedWon` (comma-separated, no decimals) |

```swift filename="StockOrderBookView.swift"
// Ask side (sellers) — blue, sorted high to low
ForEach(orderBook.asks.reversed(), id: \.price) { level in
    OrderBookRow(level: level, side: .ask, maxQty: maxQuantity, prevClose: prevClose)
}

// Bid side (buyers) — red, sorted high to low
ForEach(orderBook.bids, id: \.price) { level in
    OrderBookRow(level: level, side: .bid, maxQty: maxQuantity, prevClose: prevClose)
}
```

</AccordionItem>

### StockInfoEditSheet <KoreanLabel>통계 편집 시트</KoreanLabel>

Bottom sheet for customizing which statistics appear in the grid. Two sections: "선택 항목" (selected, with remove buttons and drag handles) and "미선택 항목" (unselected, with add buttons). Persisted via `StatisticPreferences`.

Bottom sheet showing the definition and current value of each statistic. Has horizontally scrollable pill tabs with `matchedGeometryEffect` for selecting statistics. Shows label, formatted value, and Korean explanation text.
