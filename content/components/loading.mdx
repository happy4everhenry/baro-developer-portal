<InfoBox type="fun" title="What are skeleton screens?">
Skeletons are like gray outlines that show where content will appear -- like a coloring book page before you fill it in! They shimmer gently to let you know the app is working on loading the real stuff.
</InfoBox>

## How Loading Works

Each data section on the home screen has a paired skeleton view. Skeletons are built from `SkeletonBlock` primitives with the `.shimmer()` modifier applied at the container level.

<FlowDiagram>
  <FlowRow>
    <FlowNode type="service">Skeleton visible</FlowNode>
    <FlowArrow />
    <FlowNode type="vm">Data arrives (async)</FlowNode>
    <FlowArrow />
    <FlowNode type="view">Real component swaps in</FlowNode>
  </FlowRow>
</FlowDiagram>

### Conditional Rendering Tree

```
HomeView — Conditional rendering per section
  ├── banners.isEmpty ? BannerSkeletonView : BannerCarouselView
  ├── indices.isEmpty ? MarketOverviewRowSkeletonView : MarketOverviewRowView
  ├── stocks.isEmpty  ? StockListSkeletonView : StockListView
  └── news.isEmpty    ? NewsSectionSkeletonView : NewsSectionView
```

### SkeletonBlock Primitive

The basic building block for all skeletons. It is just a rounded gray rectangle.

<PropsTable>
| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `width` | `CGFloat?` | `nil` | Fixed width (nil = flexible/fill) |
| `height` | `CGFloat` | `14` | Fixed height in points |
| `cornerRadius` | `CGFloat` | `6` | Corner radius (use 50% of height for circles) |
</PropsTable>

```swift filename="SkeletonBlock.swift"
struct SkeletonBlock: View {
    var width: CGFloat? = nil
    var height: CGFloat = 14
    var cornerRadius: CGFloat = 6

    var body: some View {
        RoundedRectangle(cornerRadius: cornerRadius)
            .fill(Color(hex: "DFE2E8"))
            .frame(width: width, height: height)
    }
}
```

## Shimmer Animation Details <KoreanLabel>시머 애니메이션 상세</KoreanLabel>

The shimmer effect uses two animations running at the same time: a diagonal gradient sweep and an opacity pulse.

### Animation Layers

| Layer | Animation | Duration | Behavior |
|-------|-----------|----------|----------|
| **Gradient sweep** | `.linear` | 1.2s | Repeats forever, no reverse. Phase: `-0.5` to `1.5` |
| **Opacity pulse** | `.easeInOut` | 0.8s | Repeats forever, autoreverses. Range: `0.4` to `1.0` |

### Gradient Configuration

```swift filename="ShimmerModifier.swift"
LinearGradient(
    stops: [
        .init(color: .clear, location: max(0, phase - 0.2)),
        .init(color: .white.opacity(0.6), location: phase),
        .init(color: .clear, location: min(1, phase + 0.2)),
    ],
    startPoint: .topLeading,
    endPoint: .bottomTrailing
)
.blendMode(.sourceAtop)  // Composited over skeleton content
```

## Best Practices <KoreanLabel>모범 사례</KoreanLabel>

<InfoBox type="tip" title="Golden rules for loading states">
1. **Match dimensions exactly** -- Skeleton blocks should mirror the loaded component's frame to prevent layout shift.
2. **Apply shimmer at the container level** -- Use `.shimmer()` on the outermost skeleton view, not individual blocks.
3. **Conditional rendering** -- Use `if data.isEmpty { Skeleton() } else { RealView(data) }` for instant swap.
4. **Skeleton color** -- All blocks use `#DFE2E8` for consistency. Circles use `#E5E7EB`.
5. **Inline skeletons** -- `MarketOverviewRowView` renders its own skeletons inline for per-section loading.
</InfoBox>
