# API Reference

## What is an API?

Imagine you're at a restaurant. You don't walk into the kitchen and cook your own food — that would be crazy! Instead, you tell the **waiter** what you want, the waiter goes to the kitchen, and comes back with your food.

An **API** (Application Programming Interface) works the same way. The BARO app on your phone is *you* at the table. The **API is the waiter**. And the kitchen? That's the server where all the stock market data lives.

Our API is built with **FastAPI**, a super-fast Python framework. It supports two ways of talking:

- **REST API** — You ask a question, you get an answer.
- **WebSocket** — An open phone line that keeps sending you live updates.

<InfoBox type="tip" title="REST vs WebSocket — What's the difference?">
**REST** is like texting a friend. You send a message, wait for a reply, done.

**WebSocket** is like a phone call that stays open. Once you connect, data keeps flowing to you in real time — like live stock prices updating every second!
</InfoBox>

<InfoBox type="note" title="Live vs Mock endpoints">
Most endpoints return **live data** from the KIS Open API. The `/api/stocks/search` endpoint currently returns **mock data** as KIS does not provide a search API. Authentication endpoints (`/api/auth/*`) are fully live with SQLite-backed user storage. Content endpoints (`/api/news`, `/api/events`, `/api/banners`, `/api/themes`) serve curated seed data. Favorites (`/api/favorites/*`) are persisted per-user in SQLite.
</InfoBox>

<InfoBox type="info" title="All roads lead to KIS">
Almost every piece of stock data in BARO comes from the **Korea Investment & Securities (KIS)** API. Our FastAPI server acts as a translator — it takes the KIS data and reshapes it into a format the iOS app can easily understand.
</InfoBox>
