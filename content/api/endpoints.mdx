# REST API Reference

<PageMeta status="live" layer="backend" />

<InfoBox type="info" title="What is a REST API?">
Think of a REST API like a drive-through window. Your app pulls up and says, "I'd like the stock prices, please!" The server hands back a neatly wrapped package of data (called JSON). Every request uses a URL (like picking a menu item) and gets back data you can read right away.
</InfoBox>

<InfoBox type="tip" title="Quick setup">
All endpoints return JSON. Base URL: `http://localhost:8000`. The BARO backend proxies Korea Investment & Securities (KIS) data, so the iOS app never needs API keys.
</InfoBox>

```bash
# Development
BASE_URL=http://localhost:8000

# All responses are JSON with UTF-8 encoding
Content-Type: application/json; charset=utf-8
```

## Authentication Endpoints

These endpoints handle user registration, login, token refresh, and Apple Sign In. All auth endpoints are **live** and backed by a local SQLite database. Prefix: `/api/auth`

<ApiEndpoint method="POST" path="/api/auth/register" description="Create a new account">

Register a new user with email and password. Returns JWT access and refresh tokens.

**Request Body**

```json
{
  "email": "user@example.com",
  "password": "securePass123",
  "name": "User Name"
}
```

**Response -- 201 Created**

```json
{
  "access_token": "eyJ...",
  "refresh_token": "eyJ...",
  "expires_in": 1800
}
```

</ApiEndpoint>

<ApiEndpoint method="POST" path="/api/auth/login" description="Sign in with email and password">

Authenticate an existing user. Returns JWT access and refresh tokens.

**Request Body**

```json
{
  "email": "user@example.com",
  "password": "securePass123"
}
```

**Response -- 200 OK**

```json
{
  "access_token": "eyJ...",
  "refresh_token": "eyJ...",
  "expires_in": 1800
}
```

</ApiEndpoint>

<ApiEndpoint method="POST" path="/api/auth/refresh" description="Refresh an expired access token">

Exchange a valid refresh token for a new access/refresh token pair. The old refresh token is revoked.

**Request Body**

```json
{
  "refresh_token": "eyJ..."
}
```

**Response -- 200 OK**

```json
{
  "access_token": "eyJ...",
  "refresh_token": "eyJ...",
  "expires_in": 1800
}
```

</ApiEndpoint>

<ApiEndpoint method="GET" path="/api/auth/me" description="Get current user profile">

Returns the authenticated user's profile. Requires `Authorization: Bearer {access_token}` header.

**Request**

```bash
curl -H "Authorization: Bearer eyJ..." http://localhost:8000/api/auth/me
```

**Response -- 200 OK**

```json
{
  "id": "uuid-string",
  "name": "User Name",
  "email": "user@example.com",
  "auth_provider": "email",
  "created_at": "2026-03-19T00:00:00+00:00"
}
```

</ApiEndpoint>

<ApiEndpoint method="POST" path="/api/auth/logout" description="Revoke refresh token">

Revokes the provided refresh token. Requires `Authorization: Bearer {access_token}` header.

**Request Body**

```json
{
  "refresh_token": "eyJ..."
}
```

**Response -- 200 OK**

```json
{
  "message": "ыбЬъ╖╕ьХДьЫГ ыРШьЧИьК╡ыЛИыЛд"
}
```

</ApiEndpoint>

<ApiEndpoint method="POST" path="/api/auth/apple" description="Sign in with Apple">

Authenticate via Apple Sign In. Creates a new account if the Apple user ID is not yet registered.

**Request Body**

```json
{
  "identity_token": "apple-jwt-token",
  "full_name": "Optional Display Name"
}
```

**Response -- 200 OK**

```json
{
  "access_token": "eyJ...",
  "refresh_token": "eyJ...",
  "expires_in": 1800
}
```

</ApiEndpoint>

<InfoBox type="fun" title="What are indices and FX rates?">
An **index** (like KOSPI) is a number that tells you how the whole stock market is doing -- like a class average on a test. An **FX rate** tells you how much one country's money is worth in another country's money.
</InfoBox>

## Stock Endpoints

Stock data for listings, details, charts, and order books. Supports multi-venue routing (KRX, NXT, Combined). Prefix: `/api/stocks`

<InfoBox type="note" title="What are market venues?">
Think of venues like different stores selling the same toy. KRX is the main store, NXT is a newer store, and "combined" shows you the best price from both.
</InfoBox>

### Market Venues

| Venue | Code | REST FID_COND_MRKT_DIV_CODE | Description |
|-------|------|----------------------------|-------------|
| `combined` | `UN` | `UN` | Unified KRX + NXT (default) |
| `krx` | `J` | `J` | Korea Exchange only |
| `nxt` | `NX` | `NX` | NXT (NextRade Korea) ATS only |

<ApiEndpoint method="GET" path="/api/stocks" description="Ranked stock list">

Fetch a ranked list of stocks for the given exchange, filter, and venue. Supports pagination. Used by home screen stock carousels and explore lists.

**Parameters**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `exchange` | `string` | Optional | Exchange identifier. One of: `kospi`, `kosdaq`, `nasdaq`, `nyse`, `amex`. Default: `"kospi"` |
| `filter` | `string` | Optional | Ranking filter. One of: `marketCap`, `popular`, `volume`, `gainers`, `losers`, `dividend`. Default: `"marketCap"` |
| `limit` | `integer` | Optional | Max items to return. Range: 1--50. Default: `5` |
| `offset` | `integer` | Optional | Number of items to skip. Default: `0` |
| `venue` | `string` | Optional | Market venue: `combined`, `krx`, `nxt`. Default: `"combined"` |

**Request**

```bash
# Top 5 KOSPI stocks by market cap (default)
curl http://localhost:8000/api/stocks

# Top 10 gainers on KOSDAQ via NXT
curl "http://localhost:8000/api/stocks?exchange=kosdaq&filter=gainers&limit=10&venue=nxt"

# US NASDAQ stocks by volume
curl "http://localhost:8000/api/stocks?exchange=nasdaq&filter=volume&limit=10"
```

## Response Data Models

<InfoBox type="note" title="Field naming convention">
Field names use `snake_case` in JSON responses. The iOS `APIClient` uses `keyDecodingStrategy = .convertFromSnakeCase` for automatic `camelCase` mapping in Swift.
</InfoBox>

<Accordion>
<AccordionItem title="MarketIndex" icon="ЁЯУК" badge="Market Data">

A single market index data point (like KOSPI or NASDAQ).

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Index identifier (e.g. `"kospi"`, `"nasdaq100"`) |
| `name` | `string` | Display name |
| `flag` | `string` | Country flag emoji |
| `value` | `float` | Current index value |
| `change` | `float` | Change from previous close |
| `change_percent` | `float` | Percentage change |
| `status` | `string` | Market status: `"open"`, `"closed"`, `"pre_market"` |

</AccordionItem>
<AccordionItem title="CurrencyRate" icon="ЁЯТ▒" badge="Market Data">

FX rate against KRW -- how much one unit of foreign money costs in Korean won.

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Currency identifier (e.g. `"usd"`) |
| `country` | `string` | Country/region name |
| `code` | `string` | ISO currency code (e.g. `"USD"`) |
| `flag` | `string` | Country flag emoji |
| `rate` | `float` | Exchange rate per 1 unit in KRW |
| `change` | `float` | Change from previous day |
| `change_percent` | `float` | Percentage change |

</AccordionItem>
<AccordionItem title="StockResponse" icon="ЁЯУЛ" badge="Stock Data">

A single stock item in a list.

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Unique ID: `"{code}-{venue}"` |
| `rank` | `integer` | Ranking position in the filtered list |
| `name` | `string` | Stock name |
| `code` | `string` | Stock code (e.g. `"005930"`) |
| `exchange` | `string` | Exchange: `kospi`, `kosdaq`, `nasdaq`, `nyse`, `amex` |
| `market_cap` | `string` | Formatted market cap (e.g. `"367ьб░ 7,183ьЦ╡"`) |
| `price` | `integer` | Current price in local currency units |
| `change` | `integer` | Price change from previous close |
| `change_percent` | `float` | Percentage price change |
| `volume` | `integer` | Trading volume |
| `dividend_yield` | `float` | Annual dividend yield (%) |
| `venue` | `string` | Trading venue: `combined`, `krx`, `nxt` |
| `is_nxt` | `boolean` | Whether this price is from NXT venue |

## Rate Limits & Notes

<InfoBox type="warning" title="Important limits to know">
The backend rate-limits KIS API calls using an asyncio semaphore (15 concurrent requests). KIS allows approximately 20 requests/second. All KIS API calls have a 15-second timeout.
</InfoBox>

<StatGrid columns={3}>
  <StatCard value="15" label="Max concurrent requests" />
  <StatCard value="~20/s" label="KIS rate limit" />
  <StatCard value="15s" label="Request timeout" />
</StatGrid>

- **KIS Upstream Limits:** 15 concurrent requests via asyncio semaphore. KIS allows ~20 req/sec.
- **Timeout:** 15-second timeout on all KIS calls. If KIS is slow or unreachable, the backend returns `502 Bad Gateway`.
- **CORS:** All origins allowed during development (`allow_origins=["*"]`). Will be restricted in production.
- **WebSocket Subscription Limit:** KIS supports ~40 real-time subscriptions per WebSocket session. The `StreamManager` uses reference counting to share subscriptions across multiple iOS clients.
