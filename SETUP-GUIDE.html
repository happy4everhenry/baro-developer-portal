1 Prerequisites

Before you begin, make sure you have the following installed on your Mac.

Node.js (v18 or later)

This project uses Next.js 16 which requires Node.js 18+. We recommend Node 20 LTS or later.

Check if Node is installed:

Terminal
node --version
# Expected: v18.x.x or higher (v20+ recommended)

npm --version
# Expected: 9.x.x or higher

Install Node.js if not present

Choose one of these methods:

Option A — Homebrew (recommended)

Terminal
# Install Homebrew if you don't have it
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Node.js LTS
brew install node@20

Option B — Official installer

Download the macOS .pkg installer from nodejs.org — choose the LTS version.

Option C — nvm (Node Version Manager)

Terminal
# Install nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# Restart your terminal, then:
nvm install 20
nvm use 20
ℹ️

Apple Silicon Macs (M1/M2/M3/M4): Node.js works natively on ARM. No Rosetta needed. Homebrew installs to /opt/homebrew/ on Apple Silicon.

Git (optional but recommended)

macOS ships with Git via Xcode Command Line Tools. Verify:

Terminal
git --version
# If not installed, macOS will prompt you to install Xcode CLI Tools

Text Editor

Any editor works. We recommend VS Code with the MDX extension for syntax highlighting of .mdx files.

2 Get the Project Files

The developer portal lives inside the BARO monorepo at developer-portal/.

If you have the project folder already

Terminal
# Navigate to the developer portal directory
cd /path/to/BARO/developer-portal

If cloning from a Git repository

Terminal
# Clone the repo (replace URL with your actual repo URL)
git clone https://github.com/your-org/baro.git
cd baro/developer-portal

Verify the directory structure

You should see these files and folders:

Terminal
ls -la

app/
components/
content/
styles/
public/
mdx-components.jsx
next.config.mjs
package.json
package-lock.json
.gitignore
⚠️

Important: Make sure you are inside the developer-portal/ directory, not the root BARO/ directory. All commands below should be run from within developer-portal/.

3 Install Dependencies

Install all required npm packages. This project has 5 dependencies:

Terminal — inside developer-portal/
npm install

# This will install:
#   next@16.x        — React framework with App Router
#   nextra@4.x       — Docs engine (MDX content layer)
#   nextra-theme-docs — Sidebar, TOC, search, dark mode
#   react@19.x       — React library
#   react-dom@19.x   — React DOM renderer

This typically takes 30–90 seconds depending on your network speed. The node_modules/ folder will be created with approximately 200–300 MB of packages.

✅

Success indicator: You should see output ending with added XXX packages and no ERR! messages. Peer dependency warnings are normal and can be safely ignored.

If you encounter permission errors

Terminal
# Fix npm permissions (avoid using sudo with npm)
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'

# Add to your shell profile (~/.zshrc for Zsh, the default on macOS):
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc
source ~/.zshrc

# Then retry
npm install

4 Start the Development Server

Launch the local development server with hot-reload:

Terminal — inside developer-portal/
npm run dev

# This runs: next dev --turbopack
# Turbopack is Next.js's Rust-based bundler for fast compilation

You should see output like:

Expected Output
  ▲ Next.js 16.1.6 (Turbopack)
  - Local:   http://localhost:3000
  - Network: http://192.168.x.x:3000

  ✓ Starting...
  ✓ Ready in 2.5s

Open in your browser

Navigate to:

Browser
http://localhost:3000

You should see the BARO Developer Docs landing page with the jade green theme, sidebar navigation, and the BARO logo.

🔄

Hot Reload: Any changes you make to .mdx files, components, or CSS will automatically refresh in the browser. No need to restart the server.

Stop the server

Press Ctrl + C in the terminal to stop the development server.

Port already in use?

If port 3000 is busy, Next.js will automatically try 3001, 3002, etc. Or specify a port manually:

Terminal
# Run on a custom port
npx next dev --turbopack -p 4000

# Or kill whatever is using port 3000
lsof -ti :3000 | xargs kill -9

Content Directory Convention

Every subfolder in content/ has a _meta.js file that controls sidebar ordering and labels:

content/screens/_meta.js (example)
// Sidebar order and labels for Screen Definitions
export default {
  'home':             'Home',
  'stock-detail':     'Stock Detail',
  'market-overview':  'Market Overview',
  'search':           'Search',
  'intelligence-hub': 'Intelligence Hub',
  'trading-floor':    'Trading Floor',
  'settings':         'Settings',
  'login':            'Login',
}

The key (e.g., 'home') must match the MDX filename (home.mdx). The value is the display label shown in the sidebar.

8 Adding or Editing Content

Edit an existing page

Simply open any .mdx file in content/, make changes, and save. The browser will hot-reload automatically.

content/getting-started/quick-start.mdx
# Quick Start

This is a standard MDX file. You can use **Markdown** syntax
plus any custom component:

<StatusBadge status="live">PRODUCTION</StatusBadge>

```swift filename="Example.swift"
let app = BaroApp()
```

Add a new page

  1. Create a new .mdx file in the appropriate content/ subdirectory:
    Terminal
    touch content/getting-started/deployment.mdx
  2. Add content to the file with a heading:
    content/getting-started/deployment.mdx
    # Deployment Guide

Instructions for deploying to Vercel...
  3. Register the page in the section's _meta.js:
    content/getting-started/_meta.js
    export default {
  'architecture': 'Architecture',
  'quick-start':  'Quick Start',
  'deployment':   'Deployment',   // ← new entry
}

The page will immediately appear in the sidebar navigation.

Add a new section (category)

  1. Create a new folder in content/
  2. Add a _meta.js file inside it
  3. Register the folder in the root content/_meta.js
  4. Add .mdx files for each page

9 Troubleshooting

Problem Solution
npm install fails Delete node_modules/ and package-lock.json, then retry: rm -rf node_modules package-lock.json && npm install
Port 3000 in use Kill the process: lsof -ti :3000 | xargs kill -9 or use a different port: npx next dev --turbopack -p 4000
"Module not found" error Run npm install again. If the error references a content file, verify the filename matches the key in _meta.js
"Element type is invalid: expected a string but got: undefined" This means a component is undefined. Check that mdx-components.jsx imports from nextra-theme-docs (NOT nextra/mdx-components)
"_meta.js refers to a page that cannot be found" Every key in _meta.js must have a matching .mdx file. Create the missing file or remove the key
Styles look wrong / no jade green Make sure styles/globals.css is imported in app/layout.jsx after the Nextra theme CSS import
"Init git repository failed" warning Harmless. Run git init inside developer-portal/ if you want to eliminate it. Does not affect functionality
Node version too old Next.js 16 requires Node 18+. Upgrade via brew upgrade node or nvm install 20
Slow first page load Normal on dev server — Turbopack compiles pages on-demand. Subsequent loads are instant. Production build (npm run build) pre-renders everything
Search not working Nextra's search requires a production build for full indexing. In dev mode, basic search is available via the search bar (top right or Cmd+K)

Nuclear reset

If everything is broken, start fresh:

Terminal — inside developer-portal/
# Remove all generated files and dependencies
rm -rf node_modules .next package-lock.json

# Reinstall everything
npm install

# Restart the dev server
npm run dev
